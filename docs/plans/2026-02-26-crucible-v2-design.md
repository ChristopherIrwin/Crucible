# Crucible V2 — Design Document
_2026-02-26_

## Overview

V2 is a cohesive upgrade across three pillars: polished dark SaaS UI, richer AI analysis, and optional Auth.js accounts with history. Anonymous use remains fully supported — auth is additive.

---

## Pillar 1: UI Overhaul

### Typography
- Swap Inter for Geist (display headings) + Inter (body)
- Larger, tighter section title hierarchy
- "CRUCIBLE" wordmark in tracked uppercase with a custom SVG forge icon

### Cards & Visual Design
- Each analysis section becomes a card with subtle border glow:
  - Tear Down: red tint
  - Rebuild: green tint
  - Scorecard: indigo tint
  - Others: neutral zinc
- `backdrop-blur` glass effect over a dark gradient background
- Landing state: subtle animated background (slow-moving gradient mesh or grid)

### Animations
- Sections slide up + fade in as they stream (not just fade)
- Pulsing ring on the Analyze button while loading
- Card entrance staggered by 100ms per section

### Scorecard
- Large score digits with color bands (red ≤4, amber 5–6, green ≥7)
- Minimal spider chart (existing Recharts radar, restyled)
- Labelled score bars beneath the chart

## Pillar 2: Richer Analysis

### Regenerate Per Section
- Each section card has a `↺` icon (top-right, visible on hover)
- Click sends `POST /api/regenerate` with `{ section, idea, intensity }`
- Response streams replacement text into the card in place
- No full re-run needed

### Follow-up Questions (Dig Deeper)
- After stream completes, a `Dig Deeper` block appears with 3 probing questions
- Questions are generated by the AI as part of the response
- Clicking a question sends a follow-up prompt, streams answer inline below the block

### Competitor / Market Landscape Section
- New `## LANDSCAPE` section added to AI output format (between Rebuild and Mutations)
- Model names 2–3 real or likely competitors and gives a rough market framing
- Parsed as a new card: `LandscapeSection`
- Prompt updated to include LANDSCAPE in the required output structure

### Confidence Badges
- Each section header shows `HIGH` / `MED` / `LOW` badge
- Comes from the AI: scorecard JSON extended with a `confidence` object:
  ```json
  {
    "clarity": 7,
    "realPain": 8,
    ...,
    "confidence": {
      "tearDown": "high",
      "assumptions": "med",
      "failureModes": "high",
      "rebuild": "med",
      "landscape": "low"
    }
  }
  ```
- Parser extracts `confidence` from scorecard JSON
- `ParsedSections` and `Scorecard` types extended accordingly

## Pillar 3: Auth.js + History

### Auth Provider
- Auth.js v5 (next-auth@beta) with GitHub + Google OAuth
- JWT sessions — no database needed for session storage
- `AUTH_SECRET`, `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` env vars

### Data Storage
- Authenticated saves stored in Vercel KV:
  - Full result: `share:{id}` (same as anonymous, but no TTL)
  - User history index: `user:{userId}:history` — sorted set keyed by timestamp, value = `{id, ideaSnippet, overallScore, createdAt}`
- Anonymous saves retain 30-day TTL (unchanged)

### New Routes
```
/dashboard          User history grid (requires auth)
/api/auth/[...nextauth]   Auth.js handler
/api/regenerate     POST — regenerate a single section
```

### UI — Sign-in Flow
- `AuthButton` component (top-right of header): shows avatar + "Sign out" when authenticated, "Sign in" when not
- After stream completes, anonymous users see a dismissible banner: *"Sign in to save your history"* with GitHub/Google buttons
- No gating — the tool never requires auth

### UI — History Sidebar
- `Clock` icon in header opens a slide-out `HistoryPanel`
- Shows last 20 analyses for authenticated users
- Each row: idea truncated to 60 chars + overall score chip + relative date
- Click navigates to `/share/{id}`

### UI — Dashboard `/dashboard`
- Grid of analysis cards for authenticated users
- Each card: idea snippet, 6 score chips (color-coded), `Re-analyze` button
- `Re-analyze` navigates to `/` with the idea pre-filled in the input

## Updated Type Definitions

```ts
interface Confidence {
  tearDown: 'high' | 'med' | 'low'
  assumptions: 'high' | 'med' | 'low'
  failureModes: 'high' | 'med' | 'low'
  rebuild: 'high' | 'med' | 'low'
  landscape: 'high' | 'med' | 'low'
}

interface Scorecard {
  clarity: number
  realPain: number
  defensibility: number
  monetization: number
  executionComplexity: number
  timing: number
  confidence: Confidence    // new
}

interface AnalysisResult {
  // ... existing fields ...
  landscape: string         // new
  followUpQuestions: string[] // new (3 questions)
}

interface ParsedSections {
  // ... existing fields ...
  landscape?: string        // new
  followUpQuestions?: string[] // new
  confidence?: Confidence   // new
}
```

## V2 Scope Boundaries

**In scope:**
- Full UI overhaul (typography, cards, animations, wordmark)
- Regenerate per section
- Follow-up questions
- Landscape section
- Confidence badges
- Auth.js with GitHub + Google
- History sidebar + dashboard

**Out of scope (V3+):**
- Idea comparison / side-by-side
- Team/org accounts
- Export to PDF
- Email notifications
- Public profile pages
